Self-Serve Launch (RCFD-PCG)

TL;DR

  • Install: pip install rcfd-pcg
  • Demo: python -m rcfd_pcg.cli demo --out results/demo_report.json --html results/demo_report.html
  • Run on your files: rcfd-pcg run --A mm:matrix.mtx --b b.npy --json --out report.json --html report.html
  • Recommend params: rcfd-pcg --d 200 --sparse --density 0.05 --recommend --json

Install (no integration required)

bash pip install rcfd-pcg

CLI usage (files-in, JSON/HTML-out)

  • Matrix A:
  • Matrix Market: --A mm:path/to/matrix.mtx (or .mtx.gz)
  • CSR NPZ: --A npz:path/to/csr.npz (with data/indices/indptr/shape)
  • Dense NumPy: --A path/to/A.npy
  • Vector b: --b path/to/b.npy

Examples: ```powershell

One-off demo with artifacts

python -m rcfd_pcg.cli demo --out results/demo_report.json --html results/demo_report.html

Run on your files (Matrix Market + NumPy vector)

rcfd-pcg run --A mm:path/to/matrix.mtx --b path/to/b.npy --json --out report.json --html report.html

Automatic parameter recommendations by regime

rcfd-pcg --d 200 --sparse --density 0.05 --recommend --json ```

Reuse preconditioner to accelerate repeated solves

```powershell

Build and save once

rcfd-pcg --n 20000 --d 200 --sparse --eps 0.25 --tol 1e-6 --save-precond pre.npz --json

Load and reuse

rcfd-pcg --n 20000 --d 200 --sparse --eps 0.25 --tol 1e-6 --load-precond pre.npz --json ```

Docker (optional, zero-install CPU)

Requires Docker Desktop running. powershell docker build -t rcfd-pcg:cpu -f Dockerfile . docker run --rm -v ${PWD}:/work -w /work rcfd-pcg:cpu ^ python -m rcfd_pcg.cli demo --out results/demo_report.json --html results/demo_report.html

Colab (optional)

Open notebooks/colab_cli_demo.ipynb to run the demo and rcfd-pcg run in the cloud.

Publishing (maintainers)

1) Bump version in pyproject.toml and update README.md changelog. 2) Build: bash python -m pip install --upgrade build twine python -m build 3) Upload to PyPI: bash twine upload dist/*

Support

  • Environment snapshot: rcfd-pcg doctor --json
  • Troubleshooting: docs/Troubleshooting.md
  • Windows consoles: prefer --json or --ascii-default.
