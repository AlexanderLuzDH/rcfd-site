Privacy

  • No telemetry by default. CLI and APIs run locally and write artifacts to your filesystem only.
  • Optional, opt-in telemetry is available for private deployments. It is disabled unless you set BOTH:
  • RCFD_TELEMETRY_ENABLE=1
  • RCFD_TELEMETRY_URL=https://your.endpoint/collect
  • Optional: RCFD_TELEMETRY_TOKEN=... (sent as Authorization: Bearer ...)
  • What is sent (opt-in only):
  • Event name, timestamp, runtime environment summary (Python version, OS name/arch), session UUID
  • High-level run properties (e.g., n, d, sparse, epsilon, aggregate timings, bail-out flag)
  • No file contents, paths, dataset names, or PII are collected by default
  • Fail-safe: network errors are ignored; telemetry never blocks or crashes the CLI.
  • Hosted playground (when enabled) publishes an explicit API spec, size/time limits, and error codes; no other data is collected.
  • Offline options:
  • CPU Docker image for air-gapped environments
  • Reproducibility packs include only environment metadata and artifacts you generate
